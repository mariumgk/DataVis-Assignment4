Dataset 1 — HR Dataset

Tree Visualizations

Source: IBM HR Analytics — HRDataset_v14.csv (311 employees, 36 columns)
The data has a natural strict hierarchy making it ideal for tree visualization.

ElementRepresents
NodesThe company root, each department, and each job position
EdgesParent-child containment — a Department belongs to the Company; a Position belongs to a Department
HierarchyCompany → Department → Position
Layout 1 · Interactive
Treemap
Interactive Plotly treemap. Area = headcount; colour = average salary. Click to drill down through the hierarchy.

Dataset 2 — Routes Dataset

Graph Visualizations

Source: OpenFlights.org — routes.csv (67,664 rows; 600-row subset used for performance)
Airports connect in a non-hierarchical many-to-many directed network, making it ideal for graph visualization.

ElementRepresents
NodesAirports identified by their 3-letter IATA code (e.g. JFK, LHR, DXB) — 265 unique airports
EdgesDirected airline routes — an edge A → B means a flight operates from airport A to airport B — 598 routes
Graph typeDirected (DiGraph) — routes have a defined source and destination
Layout 1 · Force-Directed
Spring (Force-Directed) Layout
Fruchterman-Reingold algorithm. Hub airports self-organise to the centre. Node size = degree; colour = in-degree; arrows = direction.
NetworkXDirectedHub Detection
Open Spring Layout
Layout 2 · Circular
Circular (Shell) Layout
Airports arranged in 3 concentric rings by degree tier. Inner ring = hub airports. Reveals hub-and-spoke topology clearly.
Shell Layout3 TiersHub-and-Spoke
Open Circular Layout
